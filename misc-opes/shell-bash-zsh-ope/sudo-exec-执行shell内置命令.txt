
因为sudo是通过fork() + exec() 生成子进程执行命令的, 而内置命令没有可执行文件,
sudo 无法exec().
所以 sudo + 内置 = 水土不服.

同样的, sudo不可以执行重定向命令


1: 解决方法
sudo  sh   -c "<内置命令串>
sudo  bash -c "<内置命令>"

注意这里的sh/bash 和普通命令, 没有区别,都会生成子进程.
所以对于修改进程相关的数据依旧无能为力. (比如当前目录cd, 比如cord dump允许大小: ulimic -c)


2: 对于实在困难的,设计权限的命令:
使用 sudo  -s, 切换到root用户
